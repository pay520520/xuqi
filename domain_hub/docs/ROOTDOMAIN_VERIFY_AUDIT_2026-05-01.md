## 1. 审计范围与结论

本次基于代码静态审计（前端模板、客户端 JS、客户端动作服务、模块配置定义、队列/Worker）进行评估。

**结论（简版）**
- 当前插件**尚未实现**你提出的“根域名验证”完整闭环（创建验证任务、根域名锁、5分钟自动回收、两按钮确认删除）。
- 现有架构具备可复用能力：

## 3. 可行实现方案（推荐）

## 3.1 数据表（新增）

建议新增 `mod_cloudflare_root_verify_tasks`：
- `id`
- `client_id`
- `subdomain_id`
- `rootdomain`（索引）
- `provider`（`alidns` / `dnspod`）
- `host`（后端最终使用主机记录名）
- `txt_value`
- `ttl`
- `record_id`（远端记录 ID）
- `status`（`pending`/`active`/`completed`/`cancelled`/`expired`/`cleanup_failed`）
- `locked_until`（根域名锁窗口）
- `expires_at`
- `completed_at`
- `cancelled_at`
- `cleanup_result`
- `created_at`,`updated_at`

并发核心索引建议：
- `(rootdomain, status, locked_until)`
- `(expires_at, status)`

## 3.2 配置项（后台）

新增设置：
- `enable_rootdomain_verify`（开关）
- `root_verify_ttl`（默认 600，范围如 60~3600）
- `root_verify_alidns_host`（默认 `alidnscheck`）
- `root_verify_dnspod_host`（默认 `dnspodcheck`）
- `root_verify_host_locked`（默认开，禁止用户改主机记录名）
- `root_verify_window_seconds`（默认 300）

## 3.3 业务流程

1) 用户选择子域名 + 平台 + TXT value 并提交。
2) 后端事务：
- 解析根域名；
- 查询该根域名是否存在 `active` 且 `locked_until > now()` 任务；
- 存在则返回“请等待前方用户验证完成”；
- 不存在则创建任务并写入锁窗口。
3) 后端按平台读取固定 host（若锁死则忽略用户输入），调用 DNS API 创建 TXT。
4) 成功后写 `record_id`、`status=active`、`expires_at=now()+300s`。
5) 用户点击：
- “我已完成验证并删除记录” -> 删除 TXT，状态 `completed`，立即释放锁；
- “取消验证” -> 删除 TXT，状态 `cancelled`，立即释放锁。
6) Worker 定时回收：
- 扫描 `status=active AND expires_at < now()`；
- 尝试删除 TXT；
- 状态更新为 `expired` 或 `cleanup_failed`；
- 释放锁。

## 3.4 风险控制

- 主机记录名由后端决定，防止用户滥用写任意记录。
- TXT value 做长度与字符校验（例如 1~255，禁止控制字符）。
- 接口级限流（每用户每小时 N 次）。
- 所有失败写日志（创建失败、删除失败、回收失败）并记录 provider 响应摘要。
